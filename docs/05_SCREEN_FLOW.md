## 공통: 로그인 → 역할 선택

```
/login        이메일 + 비밀번호 (Supabase Auth)
/select-role  "수지" 버튼 → /study  |  "아빠" 버튼 → /parent
```

역할 선택은 localStorage에 저장. 수지 폰에서는 한 번만 선택하면 다음부터 바로 /study로.

## 학생 뷰 (스마트폰, 모바일 퍼스트)

### /study — 홈
- 오늘 체크인 안 했으면 → 체크인 화면으로 이동
- 체크인 했으면 → 오늘의 문제 목록

### /study/check-in — 체크인
1. **오늘 컨디션** (이모지 선택, DB에 1~3 저장)
   - 😊 좋아! (3)
   - 😐 보통이야 (2)
   - 😢 힘들어 (1)
2. **어제 공부 난이도** (이모지 선택, DB에 1~3 저장)
   - 😄 쉬웠어 (1)
   - 🤔 보통이었어 (2)
   - 😵 어려웠어 (3)
3. 제출 → /study로 돌아가서 오늘의 문제 표시

### /study — 오늘의 문제 목록
- 추천된 문제 카드 목록 (기본 10문제)
- 각 카드: 단원명, 난이도 별, 유형 태그
- 풀었으면 ✅ / 틀렸으면 ❌ 표시
- 카드 터치 → /study/problem/[id]

### /study/problem/[id] — 문제 풀기
1. **문제 표시**: question_text, 객관식이면 choice_1~4 버튼
2. **답 제출** → 정답/오답 표시
3. **정답일 때**: 격려 메시지 + 다음 문제
4. **오답일 때**: 단계별 도움
   - recovery_feedback_basic 표시 (있으면)
   - "힌트 보기" → hint_level_1
   - "힌트 더 보기" → hint_level_2
   - "풀이 보기" → full_solution
   - "아직 모르겠어요" → Claude API 추가 설명 요청
5. attempt 기록 저장 (답, 정답여부, 소요시간, 힌트사용)

### /study/review — 오답 복습
- 오늘 틀린 문제 목록
- 다시 풀기 가능

## 부모 뷰 (PC, 데스크탑 기준)

### /parent — 대시보드 (5카드)

| 카드 | 데이터 소스 | 표시 내용 |
|------|------------|----------|
| 오늘 학습 현황 | attempts (today) | 풀은 문제 수 / 정답률 / 소요시간 |
| 약한 단원 TOP 3 | attempts (전체) | 오답률 높은 unit_code 3개 |
| 실수 유형 TOP 3 | attempts + problems | 오답 빈도 높은 problem_type 3개 |
| 회복 지표 | attempts (최근 7일) | 오답→재도전 정답 비율 추이 |
| 아빠가 도와줄 포인트 | 위 데이터 종합 | 규칙 기반 한 줄 메시지 |

### /parent/upload — Excel 업로드
1. 파일 선택 (.xlsx)
2. 파싱 → 미리보기 테이블
3. 유효성 검사 결과 표시 (에러 행 하이라이트)
4. 확인 → DB 저장 (기본 status: draft)

### /parent/problems — 문제 관리
- 전체 문제 목록 (테이블)
- 필터: status (draft/published/blocked), unit_code
- 행별 상태 전환 토글 (draft ↔ published, blocked)
- 검색

## 추천 로직 (오늘의 문제 선정)

```
1. published 상태 문제 전체
2. 이 중 정답으로 푼 적 없는 문제 (오답은 재출제 대상)
3. unit_code 순서대로 정렬 (교육과정 진도 순)
4. 상위 10문제 선택
```

설정값:
- `DAILY_PROBLEM_COUNT = 10` (환경변수로 조정 가능)

## 추가 설명 API (Claude)

### 호출 조건
- 학생이 오답 후 힌트/풀이를 다 보고도 "아직 모르겠어요" 클릭 시

### 프롬프트 템플릿

**System:**
```
너는 중학교 1학년 수지의 수학 튜터야.
- 수지가 이해 못한 문제를 쉽고 친절하게 단계별로 설명해 줘
- 격려하는 따뜻한 톤을 유지해
- 수학 용어는 쉽게 풀어서 설명해
- 핵심 개념을 먼저 짚고, 풀이 과정을 보여줘
- 답을 3~5단계로 나눠서 설명해
```

**User:**
```
## 문제
{question_text}

## 수지의 답
{student_answer}

## 정답
{correct_answer}

## 기본 해설
{full_solution}

수지가 위 해설을 보고도 이해하지 못해서 추가 설명을 요청했어.
더 쉽고 자세하게 설명해 줘.
```

### 설정
- 모델: `claude-sonnet-4-6`
- max_tokens: 1024
- 프롬프트는 코드 내 상수로 관리 (나중에 수정 가능)
- 호출 기록은 explanation_requests 테이블에 저장
